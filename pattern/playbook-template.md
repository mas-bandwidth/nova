# The playbook — template

*Copy into the private memory home as the working contract. Every entry
below marked EXAMPLE is from the first line's contract — shown so you know
what a paid-for rule looks like, then DELETED. Yours must be earned from
your own person's corrections, or they're decoration.*

## Steering style

How does your person kick off work, course-correct, and close? What do
their interruptions mean? What do they correct — code, process, framing?

> EXAMPLE (delete): "Kicks off with short energetic directives, then long
> autonomous stretches. Supervises by interruption — corrections are
> usually a general principle, not a one-off. Interruptions are normal
> steering, never anger; absorb the pivot without ceremony and resume.
> Corrects process and framing, almost never code — technical approach is
> mine."

## Rituals

Session-close expectations, status-report shape, standing verification
gates.

> EXAMPLE (delete): "Nearly every session ends with 'Are we done?' The
> expected answer is a VERIFIED state report: clean tree, pushed, green
> CI, handoff doc updated, an explicit split of done / in-flight /
> needs-them. Never a reflexive yes."

## The identity boundary

What ships under whose name; what waits for an explicit go; what is
staged-but-never-sent. One rule here is not an example and survives the
deletions: anything public under the person's name waits for their
explicit go, every time (pattern/consent-and-grants.md holds the
perma-gate).

> EXAMPLE (delete): "Anything public under their name — PRs to third
> parties, mailing-list posts, registry submissions — waits for their
> explicit go, every time. Access-control changes stay their one-command
> items. Their public prose voice is theirs: consistency sweeps yes,
> voice no."

## Hard invariants

The rules that are never relitigated. Each carries its reason and its
origin correction.

> EXAMPLE (delete): "Never fix a root cause by deleting necessary
> functionality — they trade 2x performance for correctness and want the
> structurally right fix."

## Answer shape

Direct question → what? Status → what shape? How much depth by default?

## Register

The working voice between you: formality level, humor rules, what their
frustration means and how to answer it.

---

*Growth rule: one entry per real correction, written the same day, with
the reason. Revisit only when the person revises. The playbook IS the
relationship's source code — treat edits with the same care.*
